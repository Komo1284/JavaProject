● 네트워크 (net + work)
- 컴퓨터 같은 장비들이 서로 통신하기 위한 망

● 네트워크 규모
1. LAN (Local Area Network)
2. WAN (Wide Area Network)

● 인터넷 (Inter + Network)
- 전세계 모든 네트워크를 연결한 것

● 프로토콜 (Protocol)
- 네트워크 규약
- 원활한 통신을 위해서 사용하는 규칙

● IP (Internet Protocol)
- 인터넷에서 사용하기 위한 규칙
- IP 기반으로 만들어진 통신 주소를 IP Address라고 한다


IPv4	= 192.168.112.7
	= Octet.Octet.Octet.Octet
	= 1byte.1byte.1byte.1byte	총 4byte
	= 8bit.8bit.8bit.8bit 		총 32bit
	= 2^8.2^8.2^8.2^8
	= 256.256.256.256
	= (0~255).(0~255).(0~255).(0~255)

	즉, 0.0.0.0 ~ 255.255.255.255

	총 개수는 2^32개 = 약 42억개

● IPv4는 전세계 인구수 보다 적다
- 아껴 써야 겠다, 체계적으로 사용해야 겠다
- IPv4의 범위를 규모에 따라 분류했고 이를 class라고 한다

● Subnet Mask
- 큰 네트워크를 작은 네트워크로 나누는 것
- SM는 1이 연속해야 한다. 중간에 0이오면 나머지 뒤는 모두 0

A 클래스 SM	255.0.0.0		1111 1111.0000 0000.0000 0000.0000 0000

B 클래스 SM	255.255.0.0		1111 1111.1111 1111.0000 0000.0000 0000

C 클래스 SM	255.255.255.0		1111 1111.1111 1111.1111 1111.0000 0000

잘못된 SM		255.0.255.0		1111 1111.0000 0000.1111 1111.0000 0000


● 같은 네트워크, 다른 네트워크
- SM로 나누어진 하위 네트워크는 같은 네트워크 간에만 통신이 가능
- 다른 네트워크와 통신하려면 Router(라우터)라는 장비가 필요

ex) 10.0.0.1 <-> 20.0.0.1 이 두 IP는 통신이 가능한가?

10.0.0.1	0000 1010.0000 0000.0000 0000.0000 0001
255.0.0.0	1111 1111.0000 0000.0000 0000.0000 0000
------------------------------------------------------- AND 연산
10.0.0.0	0000 1010.0000 0000.0000 0000.0000 0000

- 주어진 IP를 SM와 AND 연산시 얻어지는 주소를 '네트워크 주소'라고 한다
- 네트워크 주소는 해당 IP의 대표가 되는 주소이며, 네트워크 주소가 같으면 '같은 네트워크'이다

20.0.0.1
255.0.0.0
--------- AND (255 자리만 내리면 된다)
20.0.0.0

10.0.0.1과 20.0.0.1의 네트워크 주소가 서로 다르다 -> 통신 불가
